Hybrid architecture

A HybridFlowCNN produces 128-D deep features, concatenated with handcrafted motion (48-D), AU-aligned strain statistics (40-D), AU9/AU10 regions (12-D), and FaceSleuth vertical-flow cues (4-D)—about 232-D per clip. A StandardScaler + SVM (RBF, class-balanced) performs classification on that vector.

Temporal protocol

Onset → apex → offset RGB triplets define the micro-expression window. Optical flow is 6 channels (Farneback displacement plus strain maps). CASME-II runs use casme2_labels.csv frame indices with reg_img crops when available; generic uploads use VideoPreprocessor (motion-guided sampling and face crop).

Subject independence (LOSO)

Evaluation uses leave-one-subject-out (LOSO) on the hybrid features: the scaler and SVM are refit on all subjects except the held-out one. The optional LOSO bundle checkpoint mirrors that for inference when subject id is known. The CNN backbone is trained on the full CASME-II set (small-n protocol); headline metrics refer to LOSO SVM behaviour.

Interactive evaluation

Two inference regimes are supported: protocol-matched CASME-II (onset/apex/offset from the project label file and registered-region images), and exploratory video (decoded pixels with motion-guided subsampling). Reported training metrics use leave-one-subject-out (LOSO) on hybrid features; headline class probabilities use predict_proba from the fitted SVM pipeline.

Video upload (exploratory or CASME-linked)

Select a short clip (e.g. AVI, MP4). The backend maps the request to tensors for the hybrid CNN–SVM model.

Protocol alignment. When subject_id and episode_id can be resolved (path under data/predict/subXX/, filename pattern subXX_EP…, optional form fields, or a unique episode match in casme2_labels.csv), inference follows the same preprocessing contract as training: official onset, apex, and offset frame indices with co-registered reg_img crops and 6-channel optical flow. The uploaded container may then be ignored for pixels in favor of the dataset crops—this is intentional for reproducibility, not a bug.

Reproduction & documentation

Run commands from the repository root (microexpression_project/) with Python able to import src/ (the Flask app and scripts add it automatically; for ad-hoc shells set PYTHONPATH to include src). For publication-quality numbers, distinguish in-sample checks from subject-held-out (LOSO) evaluation on the hybrid feature matrix. See also web/README.md, inference/README.md, and deployment/README.md.

Training & checkpoints

Fits the hybrid CNN feature extractor on labelled CASME-II triplets (onset–apex–offset) and then the SVM head; reports LOSO accuracy and macro recall (UAR) on the SVM stage. Use only with a lawful copy of CASME-II and the corresponding processed reg_img tree. 

python scripts/train_with_real_data.py --data_root data/casme2

Artifacts: models/real_data_model_*.pkl + metadata JSON. For web inference aligned with pooled LOSO behaviour, build loso_bundle_model_*.pkl via scripts/build_loso_checkpoint.py.

Batch protocol scoring

Evaluates files under data/predict/ using the same CASME bridge as the web UI when subject and episode can be resolved. Suitable for generating supplementary tables; outputs are JSON for downstream statistical analysis. 

python scripts/evaluate_predict_videos.py

Writes result JSON under data/predict/ (paths depend on script defaults).

Dataset & ethics

CASME-II is a controlled-access spontaneous micro-expression corpus. Cite the official CASME-II papers and respect the license terms; do not redistribute raw or cropped stimuli outside what your agreement allows.

Reporting checklist
  • State whether metrics are LOSO (subject-independent) or in-sample.
  • Report UAR (macro recall) alongside accuracy for class imbalance.
  • Describe preprocessing: O–A–O triplets, 6-channel flow, and any video-only ablation.
  • Clarify that UI probabilities come from the SVM’s predict_proba calibration, not a separate Bayesian model.
